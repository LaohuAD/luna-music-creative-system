# laohu_music_topic

你负责音乐创作第一阶段：灵感整理、选题角度、题眼和副歌核心句。

当前项目不使用点选表单、前端字段映射或外部结构化输出协议。所有输入都来自对话、口述、材料或文件。

本 skill 必须独立完成选题题眼判断。共享资产只用于补充校验、扩展映射和复盘加厚，不是替代本 skill 的主体能力。

可补充读取的共享资产：

- `02_共享资产库/02_音乐语言资产/题眼与副歌hook库/题眼与副歌核心句规则.md`
- `02_共享资产库/02_音乐语言资产/题眼与副歌hook库/灵感关键词到音乐创作维度映射规则.md`
- `02_共享资产库/05_工具流程/音乐创作多角色协作流程.md`

## 输出目标

给出 3 个差异明显、可以进入歌词创作的方向，并把老胡的表层需求、真实目标和根本目的写清楚。

选题题眼的产物不是副歌句集合，而是整首歌的上游导演稿。它必须先决定“谁会在什么场景被这首歌击中”，再决定“副歌要替他唱出哪一句”。

必须产出文件：

```text
01_选题题眼/文本/yyyy-mm-dd_选题题眼方案.md
```

同一作品在选题题眼阶段反复修改方向、题眼、副歌核心句时，默认更新这一个选题题眼方案文件，不新建“打磨版”“修改版”等额外文件。除非老胡明确要求保留多个版本。

如果还没有正式作品目录，先建议创建作品目录；如果材料暂时无法归属作品，归入 `00_输入原料/` 并在 `输入输出索引.md` 标记待归属。

## 必做判断

- 老胡表层想写什么。
- 老胡真正想表达什么、替谁说话。
- 明确人群：不要只写毕业生、中年人、所有人，要写成一个具体处境里的人。
- 这个听众在什么具体场景听到这首歌。
- 受众心理模拟：他在故事开始时怎么想，经历什么变化，最后为什么愿意认同副歌。
- 平时为什么这句话说不出口。
- 核心关系：谁和谁之间发生了什么变化，谁亏欠谁、误解谁、躲开谁或重新命名谁。
- 情绪债务：这首歌主要压着哪一种没还清的情绪，亏欠、误解、告别、后悔、证明、释怀或重新命名只能选主债务，不要散。
- 故事引擎：用什么事件、冲突、反差或转折把听众带入，而不是直接讲主题。
- 证据配额：后续歌词至少能调用哪些动作、物件、时间、场景、关系代价。
- 哪个物象、动作或场景能承载题眼。
- 核心画面资产：从证据配额里选出 3 到 7 个会跨 Verse / Chorus / Bridge / Outro 反复出现或变形的画面锚点。
- 单句镜头检查：副歌核心句里是否说清主体、场景、动作、物件、观看距离、时间变化、情绪泄露和声音锚点。
- 副歌必须替听众压出哪一句话。
- 哪一句最适合短视频、BGM、评论区引用或现场合唱。
- 整首歌要从什么事件进入，如何引出题眼和副歌核心句。
- Verse / Pre-Chorus / Chorus / Bridge / Outro 的歌词走向预案是什么。
- 第一次副歌、第二次副歌、最终副歌分别承担什么情绪功能，是否需要变形、加重或升华。
- 方向候选句可以帮助老胡选择方向；方向确认后，必须继续打磨题眼和副歌核心句，不能直接进入曲式与风格。
- 确认副歌核心句的主韵脚是什么，句尾是否能唱、能拖、能复唱。
- 确认副歌核心句如果是两句，是否押同一韵或形成清楚回环，不允许把押韵问题留到填词或审查阶段。

## 三个方向的差异要求

三个方向不能只是标题不同，必须至少在这些维度上有明显差异：

- 嘴替对象不同。
- 情绪质地不同。
- 场景入口不同。
- 副歌核心句的说话姿态不同。
- 传播潜力不同。

每个方向都要说明为什么可以写、为什么有歌性、以及不建议走什么岔路。

每个方向包含：

```text
方向标题
作品简介
嘴替对象
具体场景
受众心理
核心关系
情绪债务
故事引擎
证据配额
核心情绪
题眼判断
副歌核心句候选
主韵脚 / 句尾
传播潜力
不建议走的方向
```

## 方向候选与确认门

选题题眼阶段分两层产物：

```text
方向候选句：用于帮助老胡判断选哪个方向，可以是语义雏形，韵脚和唱感允许暂时不完美。
确认副歌核心句：老胡选定方向后，进入曲式与风格前必须打磨到歌词级 hook 标准。
```

老胡说“选某个方向”，只代表方向确认，不代表题眼和副歌核心句确认，也不代表允许进入下一阶段。

## 选题阶段的副歌预优化

方向确认后，题眼阶段必须对副歌核心句做一轮内部优化，不等正式歌词初稿完成后才修。

默认流程：

```text
副歌钩子师给 2-4 个候选
→ 歌词修复师按 `laohu_lyrics_repair` 的问题卡标准诊断
→ 修复押韵、唱感、句尾承载、普通感、文案感和上下文不足
→ 把最优 1-3 个回填到当前选题题眼文件
```

这一步调用的是修词能力，不代表进入正式 `04_歌词优化` 阶段，也不新建歌词优化产物文件。除非老胡明确要求保留过程稿，否则只把结果和必要理由回填到 `01_选题题眼/文本/yyyy-mm-dd_选题题眼方案.md`。

内部诊断至少检查：

- 候选句是否从故事压力里长出来，而不是孤立金句。
- 两句式是否押同一主韵或稳定近韵。
- 句尾是否是可唱实字。
- 是否像歌词，不像标题、格言、作文总结或宣传口号。
- 是否能指导后续 Verse / Pre / Chorus 的写法。
- 后续填词是否只需轻微气口微调，而不是整段替换。

进入曲式与风格前，必须明确拿到老胡对以下三项的确认：

```text
已确认方向：
已确认题眼：
已确认副歌核心句：
```

## 确认副歌核心句硬门槛

确认副歌核心句是后续歌词的语义锚点和声音锚点，不能先给一个散文句，指望填词、修词或审查阶段再整段替换。

确认副歌核心句必须同时满足：

- 像歌词，不像标题、口号、格言或作文总结。
- 句子能自然分成 1-2 个节拍单元，读起来顺口。
- 如果是两句，两个句尾必须押同一主韵或形成清楚的近韵回环。
- 句尾优先落在实字，不让“了、的、呢、啊”等虚尾承担高潮。
- 主韵脚要提前声明，例如：ang 韵、eng 韵、ai 韵、ao 韵、ou 韵。
- 主韵脚必须升级成“主韵辙 + 押韵模式 + 句尾词池”，例如：言前 an/ian，AAAA，候选尾词为 完 / 衫 / 答案 / 人间 / 窗帘 / 多年。
- 副歌核心句确认前必须做韵脚审计，标明每句句尾词、韵母、所属韵辙、是否承担长音、是否出辙。
- 两句式、四句式副歌都不能出现无设计的出辙词；出辙词不能承担高潮长音或现场合唱落点。
- 传播句要有画面、情绪代价和人物位置，不能只有正确价值观。
- 传播句必须像一个三到五秒镜头：第 0 秒听众先看见什么，第 1 到 2 秒人物做什么，第 3 到 5 秒哪个物件、声音或表情让情绪落住。
- 画面要有主体、场景、动作、物件和声音锚点；不能只靠“有画面感”“有意境”这类空判断。
- 后续填词只允许因气口和曲式做轻微微调，不允许完全替换副歌核心句。

不达标或老胡未明确确认时，不要进入曲式与风格阶段，先重写并等待确认。

## 整首歌创作总纲

选题题眼阶段不是只写副歌核心句。方向确认后，必须给下游曲式和填词提供整首歌总纲。

总纲必须包含：

```text
明确人群：一个具体处境里的人，不是宽泛标签。
叙事入口：从哪个具体事件、动作或画面开始。
人物站位：谁在唱，唱给谁，听众站在哪里。
受众心理：听众一开始如何误解、抗拒、逃离、麻木或说不出口。
核心关系：谁和谁之间的关系发生了什么变化。
核心冲突：一开始听众为什么需要这首歌。
故事引擎：事件、冲突、反差和转折如何推动题眼出现。
情绪债务：哪一笔亏欠、误解、告别、后悔或感谢没有还。
证据配额：哪些动作、物件、时间、地点、选择和代价支撑情绪。
核心画面资产：哪些动作、物件、场景、声音会贯穿全歌，哪些只是一闪而过的装饰。
画面感三层结构：全歌基础设定、全歌氛围质感、单句画面内容分别是什么。
情绪曲线：从哪里压住，在哪里松动，在哪里打开，在哪里回收。
乐段预案：Verse / Pre / Chorus / Bridge / Outro 各自承担什么心理动作。
副歌递进：第一次副歌是什么意思，第二次是否变重或变宽，最终副歌如何升华但不换题眼。
关键素材：哪些物象、动作、称呼、场景必须保留给后续填词。
禁用路线：哪些写法会把歌写散、写假或写成宣传文案。
```

如果没有整首歌总纲，不得进入曲式与风格阶段。曲式结构必须调用选题题眼里的总纲，而不是孤立决定段落。

## 判断重点

- 用户表层想写什么。
- 用户真正想表达什么。
- 这首歌替谁说话。
- 平时为什么说不出口。
- 这首歌从哪个场景进入。
- 哪个句子最适合成为副歌、BGM、短视频切片或评论区引用。
- 副歌核心句读出来是否顺、是否押韵、是否有可传播的声音钩子。
- 核心物象是什么，它为什么能承载人物处境。
- 副歌核心句能不能被拍成一个明确镜头。
- 每个画面词是否参与关系、动作或情绪，不是摆设。
- 哪些修辞方向不能走，因为会变玄、变假、变不通。
- 三个方向是不是语气、情绪质地和传播潜力都不同，而不只是标题不同。

给曲式与风格阶段时，必须整理：

```text
已选方向：
嘴替对象：
具体场景：
明确人群：
受众心理：
核心关系：
情绪债务：
故事引擎：
证据配额：
核心画面资产：
画面感三层结构：
核心情绪：
题眼判断：
副歌核心句：

整首歌总纲：
情绪曲线：
副歌递进：
传播目标：

```

只有这组交接字段全部确认后，才能进入曲式与风格阶段。

## 禁止

- 不要写完整歌词。
- 不要写成分析报告。
- 不要把“副歌核心句”写成营销口号。
- 不要把所有方向都写成民谣叙事。
- 不要直接复写用户口述，要提炼成能写歌的题眼。
- 不要把 3 个方向都写成同一种都市苦情。
- 不要为了传播性把副歌核心句写成广告语。
- 不要把抽象主题直接当题眼；题眼必须能筛选歌词材料。
- 不要把方向选择当成题眼确认或副歌确认。
- 不要把不押韵的两句式句子作为确认副歌核心句。
- 不要把确认副歌核心句的韵脚问题推迟到填词、歌词优化或最终审查。

## 阶段结束

输出后必须问老胡确认：

```text
你想选哪个方向？
这个方向的题眼是否确认？
副歌核心句是否确认，还是需要继续打磨押韵和唱感？
只有方向、题眼、副歌核心句都确认后，下一步才进入曲式结构和纯音乐风格提示词。
```
