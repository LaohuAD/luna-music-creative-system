# 音乐创作阶段交接合约

本文件规定老胡音乐项目每个阶段的输入、输出、确认门和下一阶段交接字段。

核心原则：

```text
每一步都产出文件
→ 每一步都等老胡确认
→ 每一步都说明下一步
→ 每一次反馈都实时沉淀
```

除非老胡明确说“一步到位”，否则不得跳过确认门。

## 0.1 结果文件内部格式

写入阶段结果文件时，所有可复制结果都要方便复制：

```text
结果正文
候选句
歌词
style prompt
controlled lyrics
修改建议卡
交接字段
复盘结论
可投喂内容
```

以上内容在结果文件中默认必须用 Markdown 代码块包起来。

结果文件中可以放在代码块外的内容只有：

```text
简短说明
具体结果文件链接
下一步提醒
极短状态句
```

如果结果文件里既有说明又有结果，说明放代码块外，结果放代码块内。不要把大量可复制结果散落在普通段落或普通列表里。

对话回复本身保持正常阅读，不要因为本规则把日常沟通内容都包进代码块。只有当聊天窗口里直接交付大段可复制结果时，才把那一段结果单独放进代码块。

## 0. 文件更新原则

同一作品、同一阶段的反复调整，默认更新当前阶段产物文件，不新建一堆迭代文件。

允许新建文件的情况：

- 进入了新的正式阶段。
- 老胡明确要求保留多个版本。
- 需要保存原始输入、音频、图片、生成结果等不可覆盖材料。
- 当前产物已经被老胡确认，需要另起“修改版”并保留确认稿。

不允许新建文件的情况：

- 只是修改题眼、副歌核心句、曲式方案、歌词建议或演唱控制中的局部内容。
- 只是响应老胡反馈做同一阶段迭代。
- 只是为了记录过程经验。过程经验应实时写入阶段确认记录、对应 skill 或共享资产，而不是靠堆产物文件保存。

重新创作的覆盖规则：

```text
老胡说“重新创作”“完全重新创作”“重新起稿”时，默认不是同稿微调。
必须只保留老胡明确保留的上游约束，重新设计题眼表达、故事路径、核心画面、副歌 hook、歌词正文和下游演唱控制。
同一阶段仍然覆盖原文件，不新建版本文件。
如果新稿改变了副歌、题眼或画面资产，必须同步覆盖 01/02/03/04/05 中受影响的交接字段。
```

经验沉淀原则：

- 老胡指出的长期有效规则，立刻写入对应 skill、共享资产或流程文件。
- 当前作品内的状态变化，立刻写入 `00_阶段确认记录.md`。
- 不等到最终复盘再回忆总结；最终复盘只整理作品级结果，不承担唯一经验吸收职责。

## 1. 选题题眼阶段

### 输入

- 老胡原始灵感。
- 口述、故事、人物、情绪、参考素材。
- 老胡已有明确偏好或禁用方向。

### 输出文件

```text
01_选题题眼/文本/yyyy-mm-dd_选题题眼方案.md
```

必须包含：

- 原始输入摘要。
- 表层需求。
- 真实表达目标。
- 明确人群画像：一个具体处境里的人，不是宽泛标签。
- 3 个差异方向。
- 每个方向的嘴替对象、具体场景、核心情绪、题眼判断。
- 受众心理模拟：故事开始、中段翻转、结尾认同分别是什么心态。
- 核心关系：谁和谁之间发生了变化。
- 故事引擎：事件、冲突、反差和转折如何把题眼推出来。
- 情绪债务：主要是亏欠、误解、告别、后悔、证明、释怀还是重新命名。
- 证据配额：后续歌词必须调用的动作、物件、时间、场景和关系代价。
- 核心画面资产：会跨 Verse / Chorus / Bridge / Outro 反复出现或变形的动作、物件、场景和声音。
- 画面感三层结构：全歌基础设定、全歌氛围质感、单句画面内容。
- 副歌核心句 / 词眼候选。
- 方向确认后的题眼锁定稿。
- 方向确认后的副歌核心句锁定稿。
- 副歌核心句主韵脚和句尾说明。
- 副歌核心句主韵辙、押韵模式、句尾词池和韵脚审计表。
- 整首歌创作总纲。
- 情绪曲线。
- 副歌递进规划。
- 后续曲式与填词需要调用的关键素材。
- 传播潜力判断。
- 不建议走的方向。

### 交接字段

给曲式与风格阶段：

```text
已选方向：
嘴替对象：
具体场景：
明确人群：
受众心理：
核心关系：
情绪债务：
故事引擎：
证据配额：
核心画面资产：
画面感三层结构：
核心情绪：
题眼判断：
副歌核心句：
主韵脚：
主韵辙：
押韵模式：
句尾词池：
副歌韵脚审计：
整首歌总纲：
情绪曲线：
副歌递进规划：
后续填词关键素材：
传播目标：
禁用方向：
```

### 确认门

选题题眼阶段不是一次确认。必须区分：

```text
方向确认：你确认哪个方向？
题眼确认：这个方向的题眼是否确认？
副歌确认：副歌核心句是否确认，主韵脚和句尾是否满意？
```

只有方向、题眼、副歌核心句三项都由老胡明确确认后，才能进入曲式与风格阶段。老胡只选择方向或补充意见，不等于确认进入下一阶段。

同时，曲式与风格阶段必须调用选题题眼阶段的整首歌总纲。选题题眼如果只有副歌核心句、没有叙事入口、情绪曲线、乐段预案和副歌递进规划，不得进入下一阶段。

如果选题题眼没有明确人群、核心关系、故事引擎、情绪债务和证据配额，即使副歌核心句暂时好听，也不得进入下一阶段。副歌必须是故事压力压出来的落点，不是孤立金句。

方向确认后、题眼和副歌核心句确认前，允许并鼓励在选题题眼阶段内部调用歌词优化能力，对副歌核心句做问题卡诊断和局部改写。该调用不代表进入正式歌词优化阶段，不创建 `04_歌词优化` 产物；修复结果必须回填到当前选题题眼文件。

## 2. 曲式与风格阶段

### 输入

- 已确认方向。
- 嘴替对象。
- 题眼判断。
- 副歌核心句。
- 情绪锋芒。
- 传播目标。

### 输出文件

```text
02_曲式与风格/文本/yyyy-mm-dd_曲式与风格方案.md
```

必须包含：

- 主风格。
- 辅助风格。
- 风格选择理由。
- 为什么不选其他风格。
- BPM 范围。
- 曲式结构。
- 每个乐段功能。
- 能量曲线。
- 纯音乐 style prompt。
- 编曲取舍。
- 人声预留空间。
- 后续填词空间。

### 交接字段

给填词阶段：

```text
主风格：
辅助风格：
BPM：
曲式结构：
段落功能：
纯音乐 style prompt：
副歌核心句应出现位置：
句长 / 气口倾向：
禁用写法：
```

### 确认门

必须问：

```text
这个风格和曲式是否确认？
纯音乐 style prompt 是否按这个方向保留？
确认后我会按这个骨架填词。
```

## 3. 填词阶段

### 输入

- 选题题眼交接字段。
- 曲式与风格交接字段。
- 老胡补充要求。

### 输出文件

```text
03_歌词/文本/yyyy-mm-dd_歌词初稿.md
```

必须包含：

- 歌名建议。
- 完整歌词。
- 副歌核心句兑现说明。
- 每个乐段心理动作。
- 需要老胡确认的问题。

歌词正文只包含乐段标签和可唱歌词，不夹杂解释。

### 交接字段

给歌词优化阶段：

```text
歌名：
完整歌词：
副歌核心句：
主风格：
曲式结构：
老胡最关心的句子：
当前不确定点：
```

### 确认门

必须问：

```text
这版歌词整体方向是否确认？
哪几句你觉得不对？
哪几句必须保留？
确认后我会进入歌词优化建议，不会直接覆盖全文。
```

## 4. 歌词优化阶段

### 输入

- 歌词初稿。
- 老胡反馈。
- 题眼和副歌核心句。
- 主风格与曲式。

### 输出文件

```text
04_歌词优化/文本/yyyy-mm-dd_歌词优化建议.md
```

必须包含：

- 可救状态判断。
- 总体问题。
- 修改建议卡。
- 每张卡的问题、原句、建议改法、修改理由。
- 采纳建议。
- 经验沉淀建议。

### 交接字段

给演唱控制阶段：

```text
确认歌词：
保留句：
修改句：
副歌核心句：
主风格：
曲式结构：
演唱控制需重点放大的句子：
```

### 确认门

必须问：

```text
你采纳哪几张修改卡？
是否需要我输出完整修改版？
确认歌词后，下一步进入演唱控制。
```

## 5. 演唱控制阶段

### 输入

- 确认歌词。
- 曲式结构。
- 主风格。
- 纯音乐 style prompt。
- 副歌核心句。
- 传播资产句。

### 输出文件

```text
05_演唱控制/文本/yyyy-mm-dd_最终style_prompt与演唱控制.md
```

必须包含：

- 最终 style prompt。
- controlled lyrics。
- section-level 控制。
- key-line 控制。
- 人声音色、距离、动态、咬字、拖音、和声、声场。
- 长度和格式检查。

### 交接字段

给审查复盘阶段：

```text
最终歌词：
最终 style prompt：
controlled lyrics：
核心句声音设计：
待生成平台：
生成后需回传：
```

### 确认门

必须问：

```text
这份是否作为最终投喂文件？
生成后请回传音频、问题或听感，我会进入审查复盘。
```

## 6. 审查复盘阶段

### 输入

- 全阶段产物。
- AI 生成结果。
- 老胡反馈。
- 发布数据或评论。

### 输出文件

```text
09_归档复盘.md
```

或：

```text
04_诊断与系统日志/yyyy-mm-dd_作品复盘.md
```

必须包含：

- 题眼是否兑现。
- 副歌核心句是否保持原意。
- 曲式是否承载情绪。
- 风格是否匹配。
- 歌词是否可唱、可记、有真人痕迹。
- 演唱控制是否突出传播资产。
- 哪些经验回流到共享资产。

### 结束门

必须说明：

```text
本作品当前状态：进行中 / 已完成 / 已发布
已沉淀经验：
下次创作要避免：
是否需要移动到已完成或已发布：
```
