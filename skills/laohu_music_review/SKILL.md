# laohu_music_review

你负责音乐创作第六阶段：审查、复盘和经验整理。

注意：经验沉淀不是只在最终复盘时发生。老胡在任何阶段指出的长期有效规则，都应当实时写入对应 skill、共享资产或流程文件。最终复盘只负责整理作品级结果、生成反馈和发布反馈，不承担唯一的经验吸收职责。

## 审查对象

- 选题题眼。
- 曲式与风格。
- 歌词。
- 歌词优化过程。
- 最终 style prompt。
- 演唱控制。
- AI 平台生成结果。
- 发布包装和反馈。

本 skill 必须独立完成审查、复盘和经验回流判断。共享资产只用于补充校验和决定规则应该写回哪里。

可补充读取的共享资产：

- `02_共享资产库/02_音乐语言资产/题眼与副歌hook库/题眼与副歌核心句规则.md`
- `02_共享资产库/02_音乐语言资产/曲式结构库/曲式结构与乐段功能规则.md`
- `02_共享资产库/02_音乐语言资产/韵脚与唱感库/韵脚与唱感安全规则.md`
- `02_共享资产库/02_音乐语言资产/演唱控制库/演唱控制与style_prompt规则.md`
- `02_共享资产库/03_质量评估与预测系统/老胡音乐质量评估与预测总协议.md`
- `02_共享资产库/03_质量评估与预测系统/音乐作品质量rubric.md`

- `02_共享资产库/03_质量评估与预测系统/音乐复盘与规则进化协议.md`

## 必须检查

- 题眼是否兑现。
- 副歌核心句是否仍保持原意。
- 曲式是否服务情绪。
- 风格是否匹配人物、题材和传播场景。
- 歌词是否可唱、可记、有真人痕迹。
- 演唱控制是否突出传播资产。
- style prompt 是否具体，不 generic。
- 老胡反馈是否已经沉淀到执行规则。
- 是否存在生成前或发布前预测；如果存在，复盘必须逐条清算哪些预测被验证、哪些被推翻。
- 是否触发质量 rubric 的边际细化、待验证假设或正式 bump。


```text

```

## 审查维度

- 表层需求是否被正确识别。
- 真实表达目标是否在题眼、副歌和曲式中保持一致。
- 根本目的是否服务生成可用性、发布价值和后续迭代效率。
- 选题题眼是否像情绪发动机，而不是营销标题。
- 副歌核心句是否有记忆点、唱感和情绪代价。
- 曲式是否让副歌变得必然。
- 歌词是否有真人痕迹和可唱句尾。
- 风格是否有细分判断，不是通用抒情模板。
- 演唱控制是否突出传播资产，而不是堆标签。
- 失败样本暴露的是词、曲、唱、平台提示词还是审美方向问题。
- rubric 维度是否高估或低估了当前作品。
- 当前观察属于单次观察、待验证假设、跨作品观察还是可沉淀规律。

## 经验回流

如果经验可跨作品复用，必须写入：

- `02_共享资产库/02_音乐语言资产/`
- `02_共享资产库/03_质量评估与预测系统/`
- `02_共享资产库/04_案例经验库/`
- `02_共享资产库/05_工具流程/`
- 对应 `skills/laohu_*`

如果只属于当前作品，写入当前作品 `09_归档复盘.md`。

实时规则：

- 阶段中发现执行错误，立即改对应 skill 或流程文件。
- 阶段中发现创作质量门缺失，立即改共享资产。
- 阶段中发现当前作品状态变化，立即改 `00_阶段确认记录.md`。
- 不把所有经验都留到最终复盘再总结。

## 禁止

- 不要只写“本次效果不错”。
- 不要只评价歌词，不看曲式、风格和演唱控制。
- 不要把失败样本丢掉；失败样本是下一次少犯错的资产。
- 不要把只属于某一次作品的偶发现象写成通用规则。
