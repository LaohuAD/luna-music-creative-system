# 经验材料吸收与本地资产更新闭环

当老胡提供参考歌、歌词文章、创作理论、生成失败样本、听后反馈、发布数据或对某句歌词的评价时，进入本闭环。

```text
原始材料归档
→ 判断材料类型
→ 提炼它对音乐创作哪一环有用
→ 判断是当前作品经验还是跨作品规则
→ 更新作品复盘或共享资产
→ 必要时更新对应 laohu_ skill
→ 在输入输出索引记录
```

材料类型：

- 参考歌 / 优秀歌词：拆宏观工艺，不保存大段原文。
- 老胡反馈：提取肯定、否定、原因和长期偏好。
- 生成失败样本：记录失败原因、模型表现、下次如何避免。
- 文章 / 理论：判断进入题眼、曲式、歌词、修词、演唱控制还是复盘规则。
- 发布反馈：记录听众记住哪一句、切片效果、评论区引用、播放和转发。

原则：

- 不只改当前歌词，要改以后怎么写。
- 老胡反馈是训练材料，不是临时抱怨。
- 失败样本是资产，不是垃圾。
- 长期有效经验必须落到共享资产或 skill。
