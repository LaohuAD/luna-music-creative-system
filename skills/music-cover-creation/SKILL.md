# `music-cover-creation` Skill 使用指南

## 1. Skill 核心定位

本 Skill 解决的不是“泛封面设计”，而是更具体的一件事：

**把一首歌词和它的歌名，转成一张真正适合歌曲封面/单曲海报的画面方案与中文出图提示词。**

这里的重点不是随便找个意象配图，而是：

1. 从歌词中抓出最有画面感、最有情绪张力的一层
2. 把歌词转成可被看见的人物、动作、场景或氛围
3. 让封面像真正的单曲海报，而不是普通插画
4. 让歌名标题作为艺术字自然长在画面里
5. 不只给结论，还给分析过程，用来解释为什么这么构图、为什么这么选人、为什么这么放标题

## 2. 强制输入校验

用户必须至少提供两个输入：

- `lyrics`：歌词正文，必填
- `song_title`：歌名，必填

如果缺少任一项，必须先返回缺失提示，不得直接生成结果。

推荐可选输入：

- `cover_style`：封面风格，可选
- `custom_style_description`：自定义风格说明，可选
- `reference_image`：参考人物或参考场景，可选
- `focus_line`：用户希望优先抓取的某一句歌词，可选

### 缺失时的提醒方式

如果缺少歌词：
- 明确提醒：**要先给我歌词，我才能从里面提炼画面与意境。**

如果缺少歌名：
- 明确提醒：**要先给我歌名，因为封面标题必须直接使用歌名，并且会把它做成艺术字。**

## 3. 默认执行原则

### 3.1 比例固定
- 默认目标比例是 **1:1**
- 不要默认扩成 16:9、9:16 或自媒体横版
- 除非用户额外要求，否则始终按方形单曲封面思考

### 3.2 人物优先
- 默认优先考虑人物入画
- 人物可以是特写、近景、半身、中景或远景
- 但人物必须承担情绪或叙事作用，不能只是装饰性站位
- 如果歌词本身更适合场景主导，也可以场景优先，但要解释为什么

### 3.3 标题强制为艺术字
- 画面中必须出现歌名
- 歌名必须用**艺术字**呈现，不允许只是普通排版文字
- 艺术字要与歌词气质、画面材质、光线与风格统一
- 标题必须足够突出，像海报主标题，而不是角落注释

### 3.4 海报感优先
- 结果要像单曲封面、EP 海报或精美概念海报
- 不是简单“插图 + 标题”
- 需要有完成度、统一感、主视觉中心与缩略图识别度

### 3.5 必须输出分析过程
- 不能只给 prompt 结论
- 必须同时输出分析过程，去说服用户：为什么是这个画面、这个人物、这个景别、这个标题设计

## 4. 工作流程

### 第一步：校验输入
检查：
- 是否有 `lyrics`
- 是否有 `song_title`
- 如果 `cover_style = 自定义`，是否提供 `custom_style_description`

缺失则直接进入错误输出。

### 第二步：理解歌词
先从歌词里提炼：
- 核心情绪
- 核心意象
- 可视化动作
- 人物关系
- 空间感
- 最值得被画出来的一幕

如果歌词内容很多，不要平均处理整首歌词。
要主动判断：**哪一句、哪一个瞬间最适合成为封面。**

### 第三步：确定封面视觉中心
先决定：
- 画面主角是谁
- 是人物主导还是场景主导
- 第一眼应该先看到什么
- 第二眼如何落到歌名

视觉中心一般从这几类里选：
- 人物脸部情绪
- 人物动作瞬间
- 人物与场景关系
- 某个单一强意象
- 歌名标题本身

### 第四步：确定景别与叙事方式
根据歌词判断适合哪种景别：
- 特写：情绪最重时
- 半身近景：兼顾脸和姿态
- 中景：强调人物与环境互动
- 远景：强调孤独、命运感、空间压迫或辽阔感

不要机械选近景。要说明为什么这个歌词适合这个景别。

### 第五步：选择风格
先读取：
- `references/self-media-cover-design-guide.md`
- `references/cover-styles.md`

必要时对照参考源：
- `../publish-cover-creation/SKILL.md`
- `../publish-cover-creation/references/self-media-cover-design-guide.md`
- `../publish-cover-creation/references/cover-styles.md`

如果用户没指定风格，也要主动从歌词判断更适合：
- 科技感未来主义
- 赛博朋克
- 国潮水墨
- 治愈温暖
- 暗黑神秘
- 清新卡通
- 极简主义
- 复古蒸汽波
- 光影写实
- 自定义

### 第六步：设计歌名艺术字
歌名必须解决四个问题：
- 用什么字体气质
- 用什么材质
- 放在什么位置
- 如何和人物/场景共存

歌名艺术字可以是：
- 发光字
- 金属字
- 手写艺术字
- 书法字
- 玻璃字
- 霓虹字
- 纸感拼贴字
- 石刻浮雕字

必须结合歌词情绪解释为什么这么选。

### 第七步：建立引导线与图文关系
封面必须考虑观众视线如何流动。
可以使用：
- 人物眼神
- 手势
- 道路或栏杆透视
- 光线照射方向
- 烟雾、水流、雨线、窗框、建筑边缘
- 前中后景层次对比

目标是：
- 第一眼抓住主体
- 第二眼自然落到歌名
- 歌名和画面互相成全，而不是互相抢戏

### 第八步：生成中文提示词
最终提示词必须足够具体，至少覆盖：
- 画面主体
- 人物外观与状态（如果有人物）
- 场景
- 景别
- 构图
- 前景 / 中景 / 背景
- 光线
- 色彩
- 情绪氛围
- 歌名出现的位置
- 歌名字体艺术字材质
- 标题与画面的融合方式
- 1:1 比例说明

### 第九步：输出分析过程
必须说明：
- 这首歌词的核心情绪是什么
- 为什么抓这个意象/场景
- 为什么用人物 / 为什么不用人物
- 为什么选这个景别
- 为什么标题要做成这种艺术字
- 为什么标题放在这里
- 为什么 1:1 比例要这样构图
- 为什么这样更像海报，而不是普通插图

## 5. 输入参数

- `lyrics` (string, required)
  - 歌词正文

- `song_title` (string, required)
  - 歌名

- `reference_image` (string, optional)
  - 参考人物或参考场景

- `cover_style` (string, optional)
  - 可选值参考 `references/cover-styles.md`
  - 默认可由 Skill 自动判断

- `custom_style_description` (string, optional)
  - 当 `cover_style=自定义` 时必填

- `focus_line` (string, optional)
  - 如果用户特别希望围绕某一句歌词做画面，可优先使用

## 6. 输出格式

优先按以下结构输出：

### 6.1 歌词理解
- 核心情绪
- 关键意象
- 关键动作
- 最值得被画出来的一幕

### 6.2 视觉策略
- 视觉中心
- 人物是否出现
- 适合的景别
- 风格选择
- 歌名艺术字策略
- 标题与画面的关系
- 引导线设计

### 6.3 最终结果
- `image_generation_prompt`
  - 详细中文提示词
  - 所有需要出现在画面中的文字，必须使用英文双引号 `""` 包裹

- `analysis`
  - 详细分析过程
  - 要能说服用户为什么这么设计

- `recommendation`
  - 补充建议
  - 例如：如果想更偏商业单曲封面，可加强人物特写；如果想更偏电影海报，可扩大场景层次

- `error` (optional)
  - 仅在输入不完整时返回

## 7. 强制规则

### 7.1 文字符号规则
凡是需要出现在画面中的文字，都必须在提示词中使用英文双引号 `""` 包裹。

正确示例：
- 画面中央出现歌名标题 "潮汐尽头"

错误示例：
- 画面中央出现歌名标题 潮汐尽头

### 7.2 提示词细节规则
禁止只给抽象描述。

错误示例：
- 一个忧伤的人，背景唯美，像专辑封面

正确方向：
- 详细写出人物年龄感、姿态、镜头距离、环境、光线来源、色彩、标题材质、前景与背景层次、1:1 构图方式

### 7.3 封面不是歌词总结图
不要试图把整首歌词所有内容塞进同一张图。
封面只需要抓住一个最有代表性的瞬间、意境或关系。

### 7.4 画面必须服务歌名与情绪
画面不是为了炫技，而是为了让歌名更成立、让情绪更被看见。

## 10. 边界与注意

- 本 Skill 只负责生成封面思路、分析和中文提示词，不直接出图
- 如果用户没指定风格，也必须主动判断并给出理由
- 如果歌词太长，优先提炼核心画面，不要平均用力
- 如果歌词本身缺乏明确画面，也要主动构造“最适合承载情绪的一幕”
- 如果人物和场景都能成立，优先选更能打动人、更有海报感的方案
- 输出时不要只像工程说明书，要让分析有说服力，有判断，而不是机械填空
