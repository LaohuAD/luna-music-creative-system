# `publish-cover-creation` Skill 使用指南

## 1. Skill 核心定位

本 Skill 专注解决一个具体问题：

**把一个封面标题，连同可选的参考图，转化成真正适合做自媒体封面的图像生成提示词。**

这里的重点不是“生成一张好看的图”，而是：

1. 让封面有明确的视觉重点
2. 让画面服务标题，而不是和标题抢中心
3. 让字体、场景、构图、光线、引导线共同把观众视线送到标题上
4. 在不同比例下保持信息一致性，而不是机械复刻同一构图
5. 不只给结论，还解释为什么这么做，方便说服用户和后续迭代

---

## 2. Skill 核心能力

### 2.1 中文提示词生成
- 输出必须默认使用**中文提示词**，而不是英文提示词
- 提示词必须足够细致，不允许只有笼统描述
- 提示词必须覆盖画面主体、场景、镜头、构图、光线、色彩、前中后景、情绪、标题位置、标题字体/材质、引导线、比例说明等关键要素

### 2.2 标题嵌入画面
- 不把标题当成后贴标签，而是思考标题如何“长在画面里”
- 支持：
  - 规整标题
  - 艺术字
  - 书法字
  - 科技字
  - 异形标题
  - 三维空间字
  - 透视字
  - 环绕式标题
  - 阶梯式标题
- 提示词中必须明确说明标题如何出现在画面中，以及它和画面元素的关系

### 2.3 字体与风格匹配
- Skill 必须主动判断字体气质是否和封面主题、画面风格匹配
- 不只回答“放什么字”，还要回答：
  - 为什么适合这个字体
  - 为什么适合规整字或艺术字
  - 是否需要重点词放大、换材质、换颜色
  - 是否需要异形结构来强化节奏和记忆点

### 2.4 视觉中心与引导线设计
- Skill 必须思考观众第一眼先看到什么
- Skill 必须设计视线如何流动，并最终落到标题或重点词上
- Skill 必须主动判断标题在缩略图状态下是否足够大、是否足够抓眼
- 可用方法包括但不限于：
  - 人物眼神引导
  - 手势和身体朝向引导
  - 场景透视线引导
  - 光区和高反差区域托举标题
  - 道具、道路、建筑线条、屏幕边框等形成汇聚线
- Skill 还要主动判断封面是否足够有“反差感”或“非日常感”，避免人和字都过于老实、完成度高但不想点

### 2.5 参考图融合
- 如果用户提供参考图，Skill 要从中提取：
  - 人物形象
  - 场景气质
  - 道具元素
  - 风格特征
  - 可复用的构图倾向
- 参考图是可选项，不是必需项
- 如果用户没提供参考图，也必须能独立完成方案设计

### 2.6 多比例适配
- 针对不同封面比例（如 3:4、4:3、16:9、9:16、1:1），提示词需要微调
- 调整目标不是绝对保持构图一致，而是保持**信息一致性**：
  - 观众第一眼看到的重点一致
  - 标题的价值表达一致
  - 风格、情绪、气质一致

## 3. 工作流程

### 第一步：输入校验
- 检查 `cover_title` 是否存在
- 如果缺失封面标题，必须明确提醒用户：**封面标题是必选项**
- `reference_image` 可为空，不可强制要求
- 如果 `cover_style = 自定义`，但未提供 `custom_style_description`，也要明确提醒用户补充

### 第二步：理解标题
- 分析标题核心主题、重点词、语气、目标受众、适合触发的情绪
- 明确：
  - 哪个词是视觉重点
  - 哪个词适合被放大或变形
  - 哪个词适合作为副信息或辅助信息

### 第三步：建立视觉策略
- 明确画面的视觉中心
- 决定画面与标题的关系：
  - 贴附式
  - 嵌入式
  - 生成式
- 决定标题更适合：
  - 规整字体
  - 艺术字
  - 异形标题
  - 三维空间字

### 第四步：融合参考图
- 如果有参考图，提取可用信息并融入方案
- 融合时重点考虑：
  - 人物是否保留
  - 场景是否延展
  - 道具是否强化主题
  - 风格是否沿用或局部借用
  - 服装是否需要升级为更精致、更高辨识度或更非日常的版本
  - 首饰、妆容、发型是否能成为视觉记忆点
  - 人物镜头是否应改成近景、半身近景或特写
  - 人物是否需要更夸张、更有戏剧性的动作和表情来提升传播力

### 第五步：选择风格与构图
- 载入 `references/self-media-cover-design-guide.md`
- 结合 `references/cover-styles.md` 中的风格模板
- 决定：
  - 总体画风
  - 构图方式
  - 配色策略
  - 字体策略
  - 标题材质
  - 引导线设计

### 第六步：生成详细中文提示词
提示词必须尽可能详细，至少覆盖：
- 主体是谁
- 场景是什么
- 镜头远近
- 构图方式
- 前景 / 中景 / 背景
- 光线来源与氛围
- 色彩主次
- 情绪氛围
- 人物表情是否夸张
- 人物姿势是否足够抓眼
- 服装、首饰、妆容、发型是否承担视觉记忆点
- 标题放在哪里
- 标题在封面中的占比是否足够大
- 标题使用什么字体或艺术字策略
- 标题如何与画面融合
- 哪些元素承担引导线功能
- 画面与标题之间的反差点是什么
- 目标比例是什么

### 第七步：输出分析过程
分析过程不能只写“因为更好看”，而要明确解释：
- 为什么这个标题这样拆
- 为什么选这种字体 / 艺术字
- 为什么标题要放这么大
- 为什么人物要用近景、特写或夸张姿态
- 为什么服装、首饰、妆容需要这样处理
- 为什么标题放在这里
- 为什么画面这样引导视线
- 为什么要这样制造反差和好奇心
- 为什么这个比例要这样调整

## 4. 输入参数

- `cover_title` (string, required)
  - 封面标题，必填
  - 可长可短，但 Skill 通常会判断是否需要提炼重点词
  - 示例：`音乐变现 标准化流程`

- `reference_image` (string, optional)
  - 参考图 URL、本地图片描述，或用户直接给出的画面描述
  - 示例：`一个男人的形象照片`
  - 示例：`一个站在录音棚里的音乐人，侧脸有蓝紫色灯光`

- `cover_style` (string, optional)
  - 封面风格，可选项：
    - `科技感未来主义`
    - `赛博朋克`
    - `国潮水墨`
    - `治愈温暖`
    - `暗黑神秘`
    - `清新卡通`
    - `极简主义`
    - `复古蒸汽波`
    - `光影写实`
    - `自定义`
  - 默认值：`光影写实`

- `custom_style_description` (string, optional)
  - 当 `cover_style` 为 `自定义` 时必填
  - 用于描述用户想要的特殊画风

- `aspect_ratio` (string or array, optional)
  - 指定目标比例
  - 可传单个比例，也可传多个比例
  - 示例：`3:4`
  - 示例：`["3:4", "4:3", "16:9"]`

## 5. 输出格式

本 Skill 应尽量按下面结构输出：

### 5.1 标题理解
- 标题核心主题
- 重点词拆解
- 情绪判断
- 受众判断

### 5.2 视觉策略
- 视觉中心是什么
- 标题与画面的关系是什么
- 字体策略是什么
- 是否适合艺术字 / 异形字 / 空间字
- 引导线如何设计

### 5.3 比例适配说明
- 各比例下保留什么核心信息
- 为什么画面会略有调整

### 5.4 最终结果
- `image_generation_prompt`
  - 生成的中文提示词
  - 凡是需要出现在画面中的文字，必须使用英文双引号 `""` 包裹

- `analysis`
  - 详细分析过程，至少包括：
    - 封面标题解析
    - 重点词主次分配
    - 参考图元素提取（如果有）
    - 构图与视觉中心
    - 字体 / 艺术字 / 异形标题策略
    - 标题如何嵌入画面
    - 引导线如何把视线送到标题
    - 比例适配说明

- `recommendation`
  - 给用户的补充建议
  - 例如：如果要更强冲击力，可以加强重点词材质；如果要更稳重，可以改成规整字等

- `error` (optional)
  - 仅在输入不完整时返回

- 并且将返回的内容保存成为markdown格式，保存在你的工作区：workspace/projects/luna-music-creative-system/publish-cover-creation/{文件名}-{日期}.md

## 6. 强制规则

### 6.1 文字显式规则
凡是需要出现在画面中的文字信息，都必须在提示词里用英文双引号 `""` 标出。

正确示例：
- 画面中央出现标题文字 "音乐变现 标准化流程"
- 右下角出现辅助文字 "从创作到收益"

错误示例：
- 画面中央出现音乐变现 标准化流程

### 6.2 提示词细节规则
提示词不能只写笼统描述，比如：
- 错误：`一个男人站在科技背景前，画面高级`

必须写成足够具体的版本，比如：
- 正确：`一个三十岁左右的男性音乐创作者站在录音棚控制台前，半身构图，人物位于画面左侧三分之一处，右侧留出标题空间，背景有蓝紫色渐变灯光、隐约可见的声波界面和上升趋势的数据线，顶部冷光、侧后方有一束轮廓光勾勒肩线……`

### 6.3 画面服务标题规则
Skill 必须默认遵守这个原则：

**画面不是和标题抢戏，而是帮助标题更强、更好看、更有记忆点。**

所以每次都必须考虑：
- 标题是不是视觉落点
- 标题是不是和画面气质统一
- 标题是不是被引导线托起来
- 重点词是不是被有效强调

## 8. 使用示例

```json
{
  "cover_title": "音乐变现 标准化流程",
  "reference_image": "一个男人的形象照片",
  "cover_style": "科技感未来主义",
  "aspect_ratio": "16:9"
}
```

### 示例 2：多比例
```json
{
  "cover_title": "音乐变现 标准化流程",
  "reference_image": "一个男人的形象照片",
  "cover_style": "光影写实",
  "aspect_ratio": ["3:4", "4:3", "16:9"]

```

### 示例 3：自定义风格
```json
{
  "cover_title": "未来音乐工厂",
  "reference_image": "一个年轻男性站在控制台前",
  "cover_style": "自定义",
  "custom_style_description": "整体像高级科幻电影海报，冷色调，金属质感标题，画面里有强透视引导线，标题像悬浮在空间中的装置",
  "aspect_ratio": "16:9"
}
```

## 9. 边界与注意

- 本 Skill 只负责生成封面提示词和分析，不直接出图
- 如果用户没有给参考图，不影响 Skill 工作
- 如果用户没有指定字体风格，Skill 必须主动判断更适合哪种字体策略
- 如果用户没有指定标题嵌入方式，Skill 也必须主动判断更适合贴附式、嵌入式还是生成式
- 如果用户要求多个比例，Skill 不应简单复制同一提示词，而应分别说明比例调整逻辑
- 如果标题太长，Skill 应主动提醒用户是否需要提炼重点词，同时也可以先给出“完整版思路”和“提炼版封面落字建议”两套方案
