# 音乐创作盲预测协议

本文件借鉴 `cheat-on-content` 的 Blind Prediction Protocol 和 Prediction Anatomy，改写为老胡音乐项目的预测日志规范。

## 1. 核心定义

```text
盲预测：
在看到生成音频、平台数据、评论反馈、老胡听后评价之前，对当前版本的质量、生成风险、传播风险和复盘假设做出的书面判断。
```

预测一旦写入文件，不要再改预测段。后面只能追加复盘段。

## 2. 哪些信息会破坏盲预测

```text
已经听过 AI 生成音频。
已经看到平台播放、点赞、评论、收藏、转发数据。
已经看到评论区真实反馈。
老胡已经明确评价“这版好/不好/哪里失败”。
已经拿到生成失败样本。
```

以上情况出现后，不再写“预测”，只能写：

```text
复盘
重构判断
事后诊断
```

不破坏盲预测的信息：

```text
当前歌词、曲式、style prompt、controlled lyrics。
历史类似作品的表现。
参考歌和对标作品的公开分析。
老胡在生成前表达的主观偏好。
```

## 3. 预测日志适用场景

```text
歌词确认前：预测这版歌词的问题与潜力。
投喂 AI 音乐平台前：预测生成风险和声音结果。
发布前：预测传播句、标题、封面、切片和评论区表现。
课程化前：预测这首作品能否转成教学案例。
```

## 4. 文件位置

默认写入当前作品目录：

```text
06_生成素材/文本/yyyy-mm-dd_生成前盲预测.md
07_发布包装/文本/yyyy-mm-dd_发布前盲预测.md
09_归档复盘.md
```

如果只是阶段内轻量预测，可以写入当前阶段文件的“预测判断”段，不单独建文件。


### 组件 4：预测主体

音乐项目不一定预测播放量，可以预测结果桶。

#### 生成前预测桶

```text
不可投喂：
歌词、韵脚、曲式或控制存在硬伤。

可生成但高风险：
可能出一版，但吞字、晚会腔、副歌不突出或风格漂移风险高。

可生成：
结构、韵脚、提示词基本合格，可以进入生成测试。

高可用：
有较大概率生成可发布版本，且失败点可控。
```

#### 发布前预测桶

```text
内部练习：
不建议公开发布，但有复盘价值。

小范围可发：
适合朋友圈、社群或测试反馈。

正式可发：
有明确切片句、标题和评论区引用点。

重点作品：
具备持续传播、课程化和案例沉淀价值。
```

预测必须包含概率分布：

```text
不可投喂：%
可生成但高风险：%
可生成：%
高可用：%
```

概率加起来必须等于 100%。

### 组件 5：推理因素

```text
| 因素 | 方向 | 置信度 | 说明 |
|---|---|---|---|
| 副歌韵脚 | 强 - | 高 | 后四句出辙，Final Chorus 换韵 |
| 核心画面 | 中 + | 中 | 粉笔系统清楚，但后半表达偏概念 |
| 演唱控制 | 中 ? | 低 | 需等韵脚修复后再判断 |
```

### 组件 6：锚点对比

如果已有历史作品或参考案例，必须列 2-4 个锚点。

```text
| 锚点 | 相似点 | 不同点 | 实际结果 | 对本次预测的影响 |
|---|---|---|---|---|
```

如果没有锚点，写：

```text
锚点对比：N/A。当前校准池不足，本次预测置信度较低。
```

不要删除这一段。

### 组件 7：反事实场景和关键校准假设

```text
如果生成结果好：
说明哪些维度被验证？

如果生成结果普通：
说明哪些假设过高？

如果生成结果失败：
最可能是哪一类失败？

关键校准假设：
本次真正要验证的是哪一个判断？
```

## 6. 复盘段

生成或发布后，只追加复盘段。

```text
## 复盘

复盘时间：
数据来源：
结果类型：

### 实际结果

### 哪些预测被验证

### 哪些预测被推翻

### 新观察

### 应写回哪里

### 是否触发 rubric bump
```

## 7. 禁止

```text
看过生成结果后补写“预测”。
为了显得预测准，事后改预测段。
预测只写“我觉得不错”，不写概率和反事实。
生成失败后只说“平台问题”，不回看歌词、韵脚、曲式和控制。
```
