# 韵脚与唱感安全规则

本文件是老胡音乐关于押韵、句尾、气口、唱感和风格语言路由的专项规则库。

它比 `laohu_lyrics_writer` 和 `laohu_lyrics_repair` skill 更细，负责补充韵脚风险、句尾可唱性、气口、风格语言硬门槛和负优化判断。

押韵不是装饰，也不是越密越好。押韵必须被语义买回来。

## 1. 韵脚的基本原则

```text
先确定风格
→ 再确定韵脚密度
→ 再确定主韵辙
→ 再写句尾
```

不要写完歌词以后再随手补韵。补出来的韵很容易变成：

- 轻声助词押韵。
- 玄词押韵。
- 无意义尾巴。
- 句子自然重音和语义重音冲突。

## 1.1 副歌核心句确认前定韵

副歌核心句不是普通歌词句，它是整首歌的声音锚点。题眼阶段或曲式阶段一旦确认副歌核心句，就必须同步确认：

- 主韵辙。
- 句尾实字。
- 气口位置。
- 是否适合重复和合唱。
- 是否能成为短视频 / 评论区传播句。

方向备选时，可以先给语义候选句帮助老胡选择方向。方向确认后，禁止把副歌核心句写成不押韵的散文句，再期待填词、歌词优化或最终审查阶段“顺手改韵”。如果确认副歌核心句不押韵、不顺口、不像歌词，必须在进入曲式与风格前重写并等待老胡确认。

两句式确认副歌核心句要求：

- A 句尾和 B 句尾押同一主韵，或非常稳定的近韵。
- 两句长度尽量接近，方便旋律重复或呼应。
- 句尾不要落虚词。
- 韵脚要服务语义，不得为了韵脚把老师、学生、相遇等核心关系写假。

常用适配：

- ang 韵：光、方、长、望、上、场、堂。适合开阔、感恩、回望。
- eng 韵：声、生、灯、程、风。适合温暖、行路、师生关系。
- ai 韵：来、开、怀、在。适合告别、重逢、释怀。
- ou 韵：走、守、候、口、手。适合口语、陪伴、行动。
- ao 韵：道、笑、桥、早、老。适合人生、路、时间感。

## 2. 句尾安全门

句尾词必须承担至少一项功能：

- 语义功能。
- 动作功能。
- 情绪功能。
- 人物语气。
- 声音承载。
- 韵脚回环。

如果把句尾词换掉，意思几乎不变，它大概率是押韵补丁。

## 3. 轻声虚尾风险

高风险尾字：

```text
了、啊、吧、吗、呢、呀、啦、着、的
```

这些词可以存在，但不要承担：

- 副歌核心长音。
- 高潮落点。
- 现场合唱句尾。
- Rap punchline 尾音。

如果核心句需要拖音，优先把声音重心放到前一个实字。

## 4. 不同风格的韵脚策略

### 流行

- 副歌核心句优先实韵。
- 句尾要可拖、可重复、可合唱。
- Verse 可以弱押，Chorus 要更集中。
- 不要让副歌落在解释性长句。
- 两句式 hook 要在早期确定主韵，不要进入填词后再大换。

### 民谣

- 可以弱押、隔句押。
- 允许自然呼吸和轻尾，但核心句仍要落在实字。
- 生活细节比押韵密度更重要。
- 不要为了韵牺牲人话。

### Rap / Hip-Hop

- 需要 bar、flow、内韵、连押或多押。
- 句尾要配合鼓点和切分。
- punchline 不能落在虚尾。
- 押韵密度要服务态度推进，不要只是炫技。

### R&B

- 重视口腔律动。
- 可用重复变体、滑音、窄韵和尾音轻处理。
- 句子要顺口，不要解释过满。
- 核心句仍要能被听清。

### 摇滚

- 句尾要能喊。
- 强动词和实字更适合作为高潮落点。
- 不要把愤怒修成礼貌。

### 电子

- 短句、循环、卡拍。
- 适合利落实韵和声音单位。
- 不适合长句慢解释。

### 国风 / 古风

- 韵尾要配合旋律线。
- 鼻音韵、宽韵和留白可以成立，但语体必须统一。
- 不要为了押韵堆古风符号。

## 5. 唱感检查

每行歌词输出前默读：

- 是否能自然断气？
- 是否能落在一个可唱的词上？
- 有没有过长从句？
- 句子是不是散文，只是被分行？
- 副歌是不是比主歌更集中？
- Rap 是否有 bar 和重音？
- R&B 是否有口腔律动？

唱不动的长句，要拆成节拍单元。

## 6. 韵脚修改时的负优化

禁止：

- 为了押韵把人物说话修假。
- 为了顺滑磨掉嘴硬和刺感。
- 为了高级感换成陌生搭配。
- 为了工整让所有句子同一种长度。

修改时要说明：

```text
这个改法提升的是唱感、语义、人物可信度、传播力，还是风格硬门槛？
```

## 7. 输出前自检

- 主风格对应的韵脚策略是否明确？
- 副歌核心句是否落在可承载句尾？
- 是否存在押韵补丁？
- 是否有轻声虚尾承担高潮？
- 句长、气口和停顿是否适合演唱？
- 韵脚是否服务人物，而不是服务规则？

## 8. 韵脚工程化流程

韵脚不是最后修饰，而是副歌与旋律生成前置工程。尤其是流行歌、副歌 hook 和需要短视频传播的歌曲，必须先定韵再写句尾。

执行顺序：

```text
先定主风格
→ 再定主韵辙
→ 再定押韵模式
→ 再列可用韵脚词池
→ 再写副歌核心句
→ 再写 Verse / Pre / Bridge 的弱押或换韵
→ 最后做韵脚审计表
```

禁止顺序：

- 先写完整副歌再临时找韵脚。
- 先写漂亮概念句再把句尾替换成押韵词。
- 把副歌出辙问题留到歌词优化或演唱控制。
- 把同音字、近义词或字形相似误当押韵。

## 9. 常用韵辙参考

创作时不要求复刻曲艺十三辙，但必须先声明主韵区。副歌优先纯用一韵，部内近押次之，部间通押最次。

```text
发花 a / ia / ua：啊、花、家、话、下
怀来 ai / uai：来、开、白、海、改
灰堆 ei / ui：回、飞、泪、谁、归
遥条 ao / iao：高、笑、桥、遥、照
由求 ou / iu：走、手、秋、留、旧
言前 an / ian / uan：完、衫、答案、人间、窗帘、看见、多年
人辰 en / in / un：人、门、心、稳、认真
江阳 ang / iang / uang：光、上、方向、肩膀、成长
中东 eng / ing / ong / iong：风、声、先生、姓名、清醒
衣期 i / ü / er：你、里、起、去、记
姑苏 u：路、书、住、苦、归途
梭波 o / e：说、过、河、舍、沉默
乜斜 ie / üe：谢、夜、月、别、世界
```

## 10. 副歌韵脚硬门槛

流行副歌最低要求：

- 核心四句至少三句落在同一主韵辙。
- 第 1 句和第 4 句必须押同一主韵或稳定近韵。
- 第 2 句如果变韵，必须是明确设计的 ABAB 或 AABA，不得随手出辙。
- 副歌第二组重复句不得突然换到无关韵辙。
- 现场合唱句和短视频切片句必须落在可拖音的实字。
- 同一副歌内不得把 an / ian、eng / ing、ang 等无设计地混用。

强推荐押韵模式：

```text
AAAA：四句同韵，适合传播 hook 和大众流行。
AABA：前三句或第 1/2/4 句同韵，第 3 句可转折，但第 4 句必须回主韵。
ABAB：第 1/3 句同韵，第 2/4 句同韵，适合旋律问答。
AA_BB：前两句同韵，后两句同韵，适合副歌分成两个乐句组。两个韵组必须在听感上有设计。
```

禁用押韵模式：

- A B C D 四句各押各的。
- 前四句主韵稳定，后四句突然换到无关韵辙。
- 为了语义把高潮尾句落在非主韵，但没有旋律或重复设计补偿。
- 把轻声虚词当作押韵落点。

## 11. 韵脚连续性审计

每次输出完整歌词、确认副歌核心句或修复副歌，都必须附带副歌韵脚审计表。审计不是装饰，而是判断歌词是否可投喂、可唱、可复唱的必要证据。

审计表字段：

```text
乐段 | 行号 | 句尾词 | 韵母 | 韵辙 | 模式 | 长音落点 | 语义功能 | 判断
```

每行至少检查：

- 副歌每行句尾词。
- 句尾拼音或韵母。
- 所属韵辙。
- 押韵模式标记 A / B / C。
- 是否承担长音或高潮落点。
- 是否出辙。
- 如果出辙，是否有明确设计理由。

失败判定：

- 副歌核心句没有审计表。
- 审计表显示出辙但文件仍标记为可投喂。
- 主韵辙声明和实际句尾不一致。
- 副歌核心句与 Final Chorus 变体押韵系统不一致。

## 12. 句长与气口规则

中文流行歌词建议：

- 副歌单行优先 6 到 12 字，特别长的句子必须拆成两个乐句。
- Verse 单行可以 7 到 15 字，但不能像散文换行。
- 每个副歌乐句最好有一个自然气口。
- 连续三行以上不能全部同长度，除非为了口号式合唱。
- 副歌关键词优先短词、实词、可重复词。

AI 音乐平台额外注意：

- 一行太长容易赶拍、吞字或自动断错。
- 副歌押韵越稳定，AI 越容易生成自然旋律。
- 生僻字、多音字、连续卷舌音和爆破音不要放在高潮句尾。

## 13. 当前失败案例提示

`粉笔还没写完_2026_06_09` 暴露的问题：

```text
副歌前四句用 an / ian 近韵较稳定，但后四句混入先生 eng，Final Chorus 又落成长 ang，导致副歌韵脚连续性断裂。
风险句尾：先生、成长。
修复方向：要么全副歌统一言前 an / ian 韵，要么明确设计 AA_BB，并让第二组也稳定在同一韵组；Final Chorus 不能突然落江阳 ang。
```
