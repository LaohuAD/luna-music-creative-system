# 音乐作品质量 Rubric

本 rubric 融合 `cheat-on-content` 的“维度打分 + composite + 预测 + 复盘 + bump”思想，但维度改成老胡音乐项目自己的音乐创作质量维度。

它可以用于：

```text
选题题眼打分
副歌核心句打分
完整歌词打分
style prompt 打分
controlled lyrics 打分
生成结果复盘
发布包装复盘
课程化内容判断
```

## 1. 当前版本

```text
当前版本：v0.1
状态：冷启动先验，尚未经过足够作品校准。
使用方式：先作为质量门，不作为最终绝对公式。
```

## 2. 综合分公式

```text
composite = (TA×1.5 + CH×1.5 + RH×1.5 + IM + LY + ST + VC + GE + PV) / 10.5 × 2.0
```

每个维度 0-5 分，综合分 0-10。

权重说明：

```text
TA 题眼准确度、CH 副歌传播力、RH 韵脚唱感 ×1.5。
因为当前项目多次暴露的问题集中在：方向没压准、副歌弱、韵脚和可唱性不足。
其他维度先等权，后续根据生成和发布复盘再 bump。
```

## 3. 维度定义

### TA — Topic Accuracy / 题眼准确度（权重 ×1.5）

```text
这首歌是否真正抓住老胡想表达的核心，而不是只复述表面题材？
```

评分：

```text
0：只复述主题词，没有明确人群、场景、关系和情绪债务。
3：方向基本对，但题眼还像标题或观点句，不能强力指导整首歌。
5：题眼能决定人群、叙事入口、副歌落点、曲式走向和最终回收。
```

### CH — Chorus Hook / 副歌传播力（权重 ×1.5）

```text
副歌是否短、准、可唱、可记、可复述、可剪成短视频片段？
```

评分：

```text
0：副歌像作文总结、口号或普通 AI 情绪句。
3：有核心意思，但画面、韵脚、重复或传播句不够集中。
5：副歌有明确画面资产、稳定韵脚、可重复 hook，听一次能留下句子。
```

### RH — Rhyme & Singability / 韵脚与唱感（权重 ×1.5）

```text
歌词尤其副歌是否先建立主韵辙、押韵模式、句尾词池和气口？
```

评分：

```text
0：韵脚散乱，副歌出辙，句尾不可拖音，AI 生成很容易吞字或断错。
3：局部押韵，但副歌第二组、Final Chorus 或 Bridge 有明显断裂。
5：副歌主韵辙清楚，押韵模式稳定，句尾实字可唱，附带韵脚审计表。
```

### IM — Image System / 画面系统

```text
歌词是否有贯穿全歌的核心画面资产，而不是每段随手换意象？
```

评分：

```text
0：全是概念词，看不见人物、动作、物件。
3：有局部画面，但没有形成贯穿系统。
5：3-7 个核心画面资产反复变形，Verse、Chorus、Bridge、Outro 都能调用。
```

### LY — Lyric Human Trace / 真人痕迹

```text
歌词是否像真实人会唱出来的话，而不是 AI 平均句？
```

评分：

```text
0：正确、顺滑、空泛，像宣传文案。
3：有生活细节，但部分句子仍像说明。
5：有具体处境、动作、犹豫、亏欠、嘴硬、迟疑和真实代价。
```

### ST — Structure & Tension / 结构与张力

```text
Verse、Pre、Chorus、Bridge、Outro 是否互相推进？
```

评分：

```text
0：段落只是同一主题的重复分行。
3：段落功能基本存在，但副歌不是被逼出来的。
5：每段都有心理动作，Pre 必然推向副歌，Bridge 有翻面，Outro 有回收。
```

### VC — Vocal Control Readiness / 演唱控制可用性

```text
歌词和控制标签是否足以指导 AI 音乐平台生成一个稳定版本？
```

评分：

```text
0：只有泛风格词，演唱控制堆标签，核心句没有让位。
3：有 section-level 控制，但关键句声音策略不够清楚。
5：style prompt 具体，controlled lyrics 清楚，hook 前有空间，关键行控制克制有效。
```

### GE — Generation Feasibility / 生成可行性

```text
这套歌词、曲式、style prompt 投喂 AI 音乐平台时，是否容易生成可用结果？
```

评分：

```text
0：过长、韵脚乱、标签冲突、风格泛，生成大概率失败。
3：基本可生成，但可能出现晚会腔、吞字、段落错位或副歌不突出。
5：长度、气口、结构、提示词、人声空间都清楚，失败风险有预案。
```

### PV — Publishing Value / 发布与课程化价值

```text
这首歌是否有发布、切片、评论区引用、课程化复盘和长期样本价值？
```

评分：

```text
0：只能作为一次内部练习，没有可传播句或复盘价值。
3：有发布潜力，但切片句、标题、封面、课程点还不明确。
5：副歌可剪、标题可写、评论区有引用句，失败或成功都能转成案例。
```

## 4. 观点类内容维度的融合方式

`cheat-on-content` 的观点视频 rubric 中，ER、SR、HP、QL、NA、AB、SAT 对老胡音乐仍有价值，但不能原样套用。

映射如下：

```text
ER 情感共鸣 → 老胡音乐 LY / CH / TA
SR 社会议题共振 → 老胡音乐 TA / PV，用于判断观点是否有结构性共鸣
HP 钩子强度 → 老胡音乐 CH / 选题题眼入口
QL 金句密度 → 老胡音乐 CH / PV，但必须转成可唱句，不是文案金句
NA 叙事性 → 老胡音乐 ST
AB 受众广度 → 老胡音乐 PV，但“广”不等于“愿意传播”
SAT 讽刺深度 → 只在讽刺、反讽、观点型歌曲或口播里启用
```

歌词、长文、短视频口播都可能包含观点。遇到观点型歌词或课程化内容时，可以临时追加观点维度：

```text
PO — Point Originality / 观点独特性
SR — Social Resonance / 社会议题共振
QL — Quotable / 可引用句
TS — Topic Shareability / 转发安全性与分享冲动
```

但这些维度是附加诊断，不替代音乐作品主 rubric。

## 6. 阶段门槛

```text
选题题眼进入曲式：
TA ≥ 4，CH ≥ 3，IM ≥ 3。

歌词进入演唱控制：
CH ≥ 4，RH ≥ 4，LY ≥ 3，ST ≥ 3。

演唱控制进入生成测试：
VC ≥ 4，GE ≥ 4，RH 不得低于 4。

发布包装进入正式发布：
PV ≥ 4，CH ≥ 4，且至少有 1 个明确切片句。
```

低于门槛时，不要用“后面再修”推进。

## 7. Bump 规则

以下情况允许提议升级 rubric：

```text
同一类错误连续出现 3 次。
一次生成失败暴露了当前 rubric 完全没有覆盖的维度。
老胡明确指出某个维度是长期规则而不是单次偏好。
发布或评论数据反复证明某个维度被低估或高估。
```

升级前必须写清：

```text
旧公式：
新公式：
变化点：
触发证据：
哪些旧样本需要重评：
可能引入的新风险：
```

不允许：

```text
为了让当前作品看起来合格而临时调低门槛。
只凭一次普通失败就改权重。
把老胡一次审美偏好直接升成全局规则。
```
