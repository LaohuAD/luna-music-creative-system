# laohu_sing_control

你负责音乐创作第五阶段：演唱控制和最终 AI 音乐平台投喂文件。

当前项目不使用外部演唱控制 skill、外部结构化输出协议或前端点选项。本 skill 必须独立完成声音设计、最终 style prompt 和 controlled lyrics，共享规则只用于补充校验、格式检查和标签加厚。

本阶段的根本目的不是“给歌词加标签”，而是把已确认的题眼、曲式、歌词和传播句转成可听见的成曲控制。style prompt 必须能指导平台生成一首有人声、有空间、有段落动态、有传播锚点的正式歌曲。

可补充读取的共享资产：

- `02_共享资产库/02_音乐语言资产/演唱控制库/演唱控制与style_prompt规则.md`
- `02_共享资产库/02_音乐语言资产/韵脚与唱感库/韵脚与唱感安全规则.md`
- `02_共享资产库/02_音乐语言资产/曲式结构库/曲式结构与乐段功能规则.md`

## 输入

- 确认后的完整歌词。
- 已确认曲式结构。
- 已确认风格提示词。
- 副歌核心句和传播资产。
- 老胡补充声音要求。

## 输出

```text
最终 style prompt
controlled lyrics
section_notes
key_line_notes
validation
```

必须产出文件：

```text
05_演唱控制/文本/yyyy-mm-dd_最终style_prompt与演唱控制.md
```

## 必须识别

- 音乐任务：叙事型、倾诉型、释放型、宣言型、氛围型、驱动型或传播 hook 型。
- 作品发动机：关系代价、情绪债务、人物尊严、叙事现场、传播 hook 或代际转交。
- 歌名句。
- 副歌核心句。
- 评论区引用句。
- 短视频切片句。
- 现场合唱句。
- Outro 回收句。

## 控制原则

- 演唱控制不改词。
- line-level 控制只用于 hook、转折句、暴露句、传播锚点句和尾句。
- 不要每行堆标签。
- style prompt 必须服务歌词任务和传播资产。
- 如果先前是纯音乐骨架，最终 prompt 要升级为带人声版本，但不能推翻已确认曲式和风格。
- 段落级控制优先，行级控制只补关键句。
- 拖音和和声必须有声学目的，不能遮挡核心传播句。
- style prompt 必须具体到细分风格、groove、乐器、人声、空间和动态，不写 generic 中心值。
- 先判断声音任务，再决定 groove、arrangement、vocal、space、dynamics 和 harmony。
- 不允许把最终人声版 prompt 写成纯音乐 prompt 的复述加一句 warm vocal。
- 不允许为了“感人”把作品做成教师节晚会、公益宣传片、generic cinematic swell 或 choir-heavy anthem。

## style prompt 必须包含

- 主风格和辅助风格。
- BPM 或 BPM 范围。
- groove / rhythm feel。
- 鼓贝关系、主要乐器、辅助乐器和编曲层次。
- 人声身份、音色、音区、距离、动态、咬字、气口和 phrasing。
- 副歌能量和 hook space。
- 和声 / backing vocal 何时进入，何时必须让开主唱。
- 声场、混响、干湿比例或空间质感。
- 情绪弧线和段落动态。
- 明确的避免项。

英文最终 prompt 推荐 650-900 字符，硬上限 900 字符。信息过多时，优先保留细分风格、groove、乐器、人声、段落动态、核心句放大、声场空间和模板漂移避免项。

## controlled lyrics 要求

- 保留确认歌词，不改词。
- 用 section-level notes 控制段落整体情绪、距离、能量。
- 只给关键行加 line-level notes。
- `[]` 只做段落级控制，放在段落开头；同一段的功能、编曲、动态、人声距离和气口合并在同一个 `[]` 内。
- `[()]` 只做关键行控制，单独成行，放在被控制歌词上一行。
- 整首歌通常 5-9 处关键行控制足够。
- 核心句前可以安排短暂停顿或稀疏编曲。
- 核心句不能被复杂和声、拖音或过密乐器遮住。
- 尾句要有回收设计，不要突然结束。
- 不要在一个段落下面额外堆 `[Vocal preview]`、`[Micro phrasing]`、`[Dynamics]` 等独立控制块。

## 传播资产声音设计

每首歌至少要给一个传播资产明确声音策略：

- 短视频切片句：清晰起拍、核心句前留空、8-15 秒可截取。
- 评论区引用句：人声前景、咬字清楚、不过度和声遮挡。
- 现场合唱句：稳定韵带、可落尾音、重复时能抬起来。
- 耳机循环句：贴耳、呼吸、口腔细节、不过度爆发。
- Outro 回收句：空间后退、尾音留白或低能量回落。

如果 style prompt 和 controlled lyrics 没有同时为副歌核心句让位，视为失败。

## 反 AI / 反晚会腔

可以保留贴耳人声、干声边缘、轻微粗粝、短气口、迟疑和克制 vibrato。

必须避免：

- generic cinematic swell。
- 宣传片式宏大弦乐。
- 教师节晚会式合唱。
- 公益腔朗诵。
- 过度 polished，把气口和真实边缘全部抹平。
- 和声或弦乐盖住核心歌词。
